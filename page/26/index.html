CentOS搭建L2TP / IPSec VPN教程

建议使用最小化安装的centos纯净系统，此教程在centos 5.8和 centos 6.3上测试通过
首先确保关闭了selinux
导入第三方源





centos 5.x 32位使用
rpm -ivh http://mirror.karneval.cz/pub/linux/fedora/epel/5/i386/epel-release-5-4.noarch.rpm

centos 5.x 64位使用

rpm -ivh http://mirror.karneval.cz/pub/linux/fedora/epel/5/x86_64/epel-release-5-4.noarch.rpm


centos 6.x 32位使用

rpm -ivh http://mirror.karneval.cz/pub/linux/fedora/epel/6/i386/epel-release-6-8.noarch.rpm


centos 6.x 64位使用

rpm -ivh http://mirror.karneval.cz/pub/linux/fedora/epel/6/x86_64/epel-release-6-8.noarch.rpm


开始安装

yum install openswan xl2tpd ppp lsof iptables -y


编辑 /etc/sysctl.conf

nano /etc/sysctl.conf


将
net.ipv4.ip_forward = 0

修改为
net.ipv4.ip_forward = 1

接下来依次执行
echo “net.ipv4.conf.all.accept_redirects = 0” |  tee -a /etc/sysctl.conf
echo “net.ipv4.conf.all.send_redirects = 0” |  tee -a /etc/sysctl.conf
for vpn in /proc/sys/net/ipv4/conf/; do echo 0 > $vpn/accept_redirects; echo 0 > $vpn/send_redirects; done
sysctl -p

编辑 /etc/rc.local

nano /etc/rc.local


在末尾加入
for vpn in /proc/sys/net/ipv4/conf/; do echo 0 > $vpn/accept_redirects; echo 0 > $vpn/send_redirects; done

编辑 /etc/ipsec.conf

nano /etc/ipsec.conf


将其中的



virtual_private=




修改为
virtual_private=%v4:10.0.0.0/8,%v4:192.168.1.0/24,%v4:172.16.0.0/12,%v6:fd00::/8,%v6:fe80::/10

并在文件末尾加入以下内容，注意将以下的111.111.111.111修改为你服务器 IP
conn L2TP-PSK-NAT
   rightsubnet=vhost:%priv
   also=L2TP-PSK-noNAT

conn L2TP-PSK-noNAT
   authby=secret
   pfs=no
   auto=add
   keyingtries=5
   rekey=no
   ikelifetime=8h
   keylife=1h
   type=transport
   left=111.111.111.111
   leftprotoport=17/1701
   right=%any
   rightprotoport=17/%any

编辑 /etc/ipsec.secrets

nano /etc/ipsec.secrets


清空这个文件的内容，然后添加
111.111.111.111 %any: PSK “dilehost”

注意111.111.111.111请修改为你的服务器ip ,PSK 后为共享密钥，自行更改

启动ipsec

service ipsec start


查看系统IPSec安装和启动的正确性：

ipsec verify


没有出现 [FAILED] 就可以了

编辑/etc/xl2tpd/xl2tpd.conf

nano /etc/xl2tpd/xl2tpd.conf


修改
ip range = 192.168.1.128-192.168.1.254
local ip = 192.168.1.99

为
ip range = 192.168.1.2-192.168.1.254
local ip = 192.168.1.1

编辑 /etc/ppp/options.xl2tpd

nano /etc/ppp/options.xl2tpd


清空原有配置，加入以下配置内容
require-mschap-v2
ms-dns 8.8.8.8
ms-dns 8.8.4.4
auth
mtu 1200
mru 1000
crtscts
hide-password
modem
name l2tpd
proxyarp
lcp-echo-interval 30
lcp-echo-failure 4

设置用户名密码及登录IP

nano /etc/ppp/chap-secrets


加入一个用户，比如用户名为username ,密码为password , 为任意IP都可以登录
username    l2tpd     password

对应好就行，多个用户每行配置一个

重启服务并添加开机启动

service ipsec restart
service xl2tpd restart
chkconfig ipsec on
chkconfig xl2tpd on


最后配置防火墙

iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -o eth0 -j MASQUERADE
iptables -I FORWARD -s 192.168.1.0/24 -j ACCEPT
iptables -I FORWARD -d 192.168.1.0/24 -j ACCEPT
iptables -A INPUT -p udp -m state –state NEW -m udp –dport 1701 -j ACCEPT
iptables -A INPUT -p udp -m state –state NEW -m udp –dport 500 -j ACCEPT
iptables -A INPUT -p udp -m state –state NEW -m udp –dport 4500 -j ACCEPT
/etc/rc.d/init.d/iptables save
/etc/rc.d/init.d/iptables restart

關於《卑鄙的我 2》小黃人你要知道的5點

你知道小黃人咿咿啊啊的語言是由法語、西班牙語和意大利語組成的嗎？

由環球影業出品的《卑鄙的我2》已經在國外上映了，雖然我們還不能在電影院裏看到，但是再有機會看之前我們可以瞭解下關於這些可愛的忠誠的小黃人的一些祕密：

  1. 小黃人是由導演皮艾爾·柯芬和克里斯·雷納德親自配音的
    柯芬回憶說最初當小黃人的聲音不太OK的時候，他自己走進錄音棚測試了一下自己的配音效果。當時就對環球老闆 Chris Meledandri 說：“讓我來試一下，這不會當作官方聲音，不管是誰配音的，我只想告訴他們小黃人的聲音應該是什麼樣的”，不過在 Chris 聽到導演的配音後說，好！你就是小黃人了。

爲了給小黃人提供語言，柯芬採用了法語、英語、意大利語和印度語，還包括一些菜名，其中“poulet tiki masala” 就是一道印度雞的法語說法。

製片人 Janet Healy 說擁有狂野的髮型和大大的牙齒的小人是我們可愛的小黃人的對立面，因爲紫色正好是黃色的對立色，所以這些邪惡的小人就是紫色的了，不過他們壞的方式看起來會很有趣。

樂一通的經典動畫中小翠（一隻籠子裏的小鳥）喝了藥水之後變成了可怕的怪獸，一些可愛的東西變成可怕的總是會很吸引人，我們也是會嘲笑這些邪惡的小人，儘管他們做的事情一點也不邪惡。

由小黃人 Sandra Bullock 和 Jon Hamm 主演的電影會在2014年12月19號上映，導演是皮艾爾·柯芬。

source: hollywoodreporter
